# <a name="prebuilt-domain-reference"></a>预构建的域参考
此参考提供有关[预构建的域](luis-how-to-use-prebuilt-domains.md)的信息，这些参考是 LUIS 提供的意向和实体的预构建集合。

相比之下，[自定义域](luis-how-to-start-new-app.md)从无意向和模型开始。 可将任何预构建的域意向和实体添加到自定义模型中。

## <a name="calendar"></a>日历 

日历域提供与日历条目相关的意向和实体。 日历意向包括添加、删除或编辑约会、检查可用性以及查找有关日历条目或约会的信息。

### <a name="intents"></a>意向
| 意向名称 | Description | 示例 |
| ---------------- |-----------------------|----|
| 添加 | 将新的一次性项添加到日历中。| 周日下午 2 点与 Lisa 有约会 <br/><br/>我希望安排一个会议<br/><br/>我需要设立一个会议|
| 检查可用性 | 在用户的日历或其他人的日历上查找可安排约会或会议的时间。| 何时可与 Jim 见面？ <br/><br/>显示 Carol 明天的空闲时间<br/><br/>Chris 周六有空吗？|
| 删除 | 请求删除日历条目。| 取消与 Carol 的约会。 <br/><br/>删除上午 9 点的会议<br/>|
| 编辑 | 请求更改现有会议或日历条目。| 将上午 9 点的会议移至上午 10 点。<br/><br/>我想更新日程安排。<br/><br/>重新安排与 Ryan 的会面。|
| 查找 | 显示我的周日历。| 预约看牙医。 <br/><br/>显示我的日历<br/>|

### <a name="entities"></a>实体
| 实体名称 | Description | 示例 |
| ---------------- |-----------------------|----|
| 位置 | 日历项、会议或约会的位置。 地址、城市和区域是位置的典型示例。| 209 纳什维尔体育馆 <br/><br/>897 薄煎饼店<br/><br/>车库|
| 主题 | 会议或约会的标题。| 与牙医的预约 <br/><br/>与 Julia 共进午餐<br/><br/>与医生的预约|
